## Phase 1 MVP 이후 필요한 마이그레이션

### Step 1 — 백엔드 서버로 이동 (Phase 2 준비)
1. FastAPI 백엔드 프로젝트 생성
2. 프론트엔드 로컬 상태 → 백엔드 데이터베이스로 마이그레이션
3. API 엔드포인트 구현
4. 인증 시스템 추가
5. Docker 기반 배포 설정

### Step 2 — Edge AI 지원 (Phase 3 준비)
1. React Native 또는 Expo로 모바일 앱 변환
2. 로컬 벡터 DB (Chroma) → SQLite 기반으로 안정화
3. 오프라인 모드 지원
4. 동기화 엔진 구현 (로컬 ↔ 서버)

### Step 3 — 다국어 지원
1. i18n 설정 추가 (현재는 영어 + 스페인어 기본)
2. 사용자 언어 설정 저장
3. AI 프롬프트 다국어 템플릿

### Step 4 — 성능 최적화
1. 임베딩 생성 성능 개선 (batch processing)
2. 벡터 검색 최적화 (인덱싱)
3. 번들 크기 최적화 (Tree-shaking, 코드 분할)
